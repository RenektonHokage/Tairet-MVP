# TAIRET — RIESGOS_ACEPTADOS_Y_PENDIENTES (V1)

## 1) Título

**TAIRET — RIESGOS_ACEPTADOS_Y_PENDIENTES (V1)**

## 2) Ruta canónica

`docs/audits/RIESGOS_ACEPTADOS_Y_PENDIENTES.md`

## 3) Propósito

Consolidar el estado actual de riesgos del MVP en cuatro grupos:

* **riesgos aceptados por negocio**
* **riesgos críticos abiertos**
* **pendientes no bloqueantes**
* **riesgos que requieren validación**

Este documento usa **código como fuente primaria** y deja explícito todo lo no confirmado.

## 4) Regla de fuente de verdad

1. **Código del repo** (fuente primaria)
2. **Runtime/entorno** solo cuando el código no alcanza para confirmar comportamiento real
3. **Input humano** solo para decisiones de negocio, riesgo aceptado o comportamiento esperado

Regla obligatoria: todo lo no demostrado queda como **`Requiere validación`**.
